1. dispel4py Commands for All the Use Cases
Note: I have previously used processor.py directly for execution for simplicity and did not compile it as a dispel4py binary.

- Group by all
RDD: 
dispel4py\new\processor.py dynamic_redis_v1 dispel4py.examples.graph_testing.grouping_onetoall_stateful -ri localhost -i 2 -n 6


FD:
dispel4py\new\processor.py dynamic_redis dispel4py.examples.graph_testing.grouping_onetoall_stateful -ri localhost -i 2 -n 6


Note that the only difference of two modes is the target. RDD is "dynamic_redis_v1" and FD is "dynamic_redis".

- Group by global

RDD:
dispel4py\new\processor.py dynamic_redis_v1 dispel4py.examples.graph_testing.grouping_alltoone_stateful -ri localhost -i 2 -n 7

FD:
dispel4py\new\processor.py dynamic_redis dispel4py.examples.graph_testing.grouping_alltoone_stateful -ri localhost -i 2 -n 7


- Group by fields

RDD: 
dispel4py\new\processor.py dynamic_redis_v1 dispel4py.examples.graph_testing.group_by_stateful -i 20 -ri localhost -n 5

FD:
dispel4py\new\processor.py dynamic_redis dispel4py.examples.graph_testing.group_by_stateful -i 20 -ri localhost -n 5


- Astrophysics
RDD: 
dispel4py\new\processor.py dynamic_redis_v1 dispel4py.examples.internal_extinction.int_ext_graph -n 4 -ri localhost -d "{\"read\" : [ {\"input\" : \"coordinates.txt\"} ]}"

FD:
dispel4py\new\processor.py dynamic_redis dispel4py.examples.internal_extinction.int_ext_graph -n 4 -ri localhost -d "{\"read\" : [ {\"input\" : \"coordinates.txt\"} ]}"

For more details, you can see https://github.com/zhtisi/dispel4py/tree/dynamic_redis_stateful_dev/dispel4py/examples/article_sentiment_analysis .

- Sentiment Analysis
RDD:
Not supported.

FD:
dispel4py\new\processor.py dynamic_redis dispel4py.examples.article_sentiment_analysis.analysis_sentiment -ri localhost -n 32 -d "{\"read\" : [ {\"input\" : \"Articles_cleaned.csv\"} ]}"

For more details, you can see https://github.com/zhtisi/dispel4py/tree/dynamic_redis_stateful_dev/dispel4py/examples/internal_extinction .

2. Environment Preparation

As introduced in last email, that attachment contains all the instructions for preparing the environment before running.

Note that all Redis-related code requires an external Redis service. If you already have one, you can specify which Redis service you need to connect to by modifying the parameters -ri and -rp (my evaluation is based on a local Redis service, a remote Redis service may affect performance to some extent). If not, you can refer to steps in the attachment or Redis document(https://redis.io/docs/getting-started/) to install it. I used version 5.0.7 in my evaluation.
